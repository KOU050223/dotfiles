以下の手順でプルリクエストを作成してください：

## 対象ブランチ

- ベースブランチ: ${1:-main}
- タイトル: ${2:-現在の変更内容から自動生成}

## 実行手順

1. **設定ファイルの確認**

   - ~/commit-lint-config.js が存在する場合は、その設定を読み込んでコミットメッセージ規則に従う
   - .github/pull_request_template.md または .github/PULL_REQUEST_TEMPLATE.md が存在する場合は、そのテンプレートを使用して PR 本文を作成
   - 上記ファイルが存在しない場合は、デフォルトのフォーマットを使用

2. **現在の状態を確認**

   - 現在のブランチを確認
   - 変更内容を確認（git status, git diff）
   - コミット履歴を確認（git log --oneline）
   - ベースブランチとの差分を確認（git diff ${1:-main}...HEAD）

3. **変更をコミット**

   - まだコミットされていない変更がある場合は、適切なコミットメッセージでコミット
   - ~/commit-lint-config.js の規則に従う（存在する場合）
   - 一般的なフォーマット例：
     - feat: 新機能追加
     - fix: バグ修正
     - docs: ドキュメント更新
     - refactor: リファクタリング
     - test: テスト追加・修正
     - chore: ビルドプロセスや補助ツールの変更
   - 必要に応じて serena などのツールを活用してコミット品質を向上

4. **リモートにプッシュ**

   - 現在のブランチをリモートにプッシュ
   - 必要に応じて upstream を設定（git push -u origin ブランチ名）

5. **プルリクエストの作成**

   - gh コマンドを使用してプルリクエストを作成
   - PR テンプレートが存在する場合はそれに従う
   - テンプレートがない場合のデフォルト構成：
     - タイトル: 変更内容を簡潔に要約
   - メッセージに Author 情報や Co-Authored-By などの署名は含めない
   - 本文:
     - ## 概要
       何を変更したか、なぜ変更したかを説明
     - ## 変更内容
       具体的な変更点を箇条書き
     - ## テスト
       テスト内容や確認事項
     - ## 関連 Issue
       関連する Issue 番号があれば記載（Closes #123 など）
     - ## スクリーンショット
       UI の変更がある場合は追加
     - ## チェックリスト
       - [ ] テストを実行して合格
       - [ ] ドキュメントを更新
       - [ ] 変更ログを更新（必要に応じて）

6. **確認事項**
   - コンフリクトがないか確認
   - CI/CD が正常に動作するか確認
   - レビュアーを適切に設定
   - ラベルやマイルストーンを設定（必要に応じて）

## 注意事項

- プルリクエストを作成する前に、必ず変更内容をレビューしてください
- コミットメッセージは意味のあるものにしてください
- 大きな変更の場合は、複数のプルリクエストに分割することを検討してください
- PR のタイトルと本文は、レビュアーが理解しやすいように明確に記述してください

## 使用例

```bash
# デフォルト（mainブランチへ、タイトル自動生成）
/pullrequest

# developブランチへのPR
/pullrequest develop

# タイトル指定
/pullrequest main "ユーザー認証機能の追加"
```
